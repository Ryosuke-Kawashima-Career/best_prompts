# User Story

## User Stories

As an international traveler, I would like to make an itinerary for my trips, making reservations of flights, hotels, restaurants, public transportation, etc. so that I can experience seamless and frictionless processes in my journey.
If some accidents happen on the trip, I would like to adjust changes to my plans and schedules as soon as possible.

## Developer Stories

As a developer of this reservation app, I would like to make the code simple and comprehensible so that the code is maintainable and scalable for the future development.
I prefer the frontend developed by TypeScript frameworks while the backend is developed by Java since this is the standard of my company.

## Function Details

1. User inputs dates and places to visit.
2. System shows candidates of reservations with the prices
3. User makes reservations or preserve them in their cart just as what they do on Amazon.com.
4. User can check the schedules and procedures necessary before and during the trip.
5. User can change the schedule when some accidents happen.

## UI

- The UI is similar to the ones developed by Online Traveling Agencies such as Expedia, Agoda, Booking.com, SkyScanner, Yeogiotte (a Korean OTA), etc.
