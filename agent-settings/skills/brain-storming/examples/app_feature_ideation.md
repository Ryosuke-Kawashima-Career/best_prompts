# Example: App Feature Ideation

## Context

The user wants to add a new "Social" feature to a travel dashboard.

---

### Phase 1: Divergent Thinking

- **Idea 1**: Real-time chat with other travelers at the same location.
- **Idea 2**: Shared "Bucket List" that can be voted on by friends.
- **Idea 3 (Wildcard)**: Augmented Reality (AR) "Digital Graffiti" that users can leave at landmarks for friends to find.
- **Idea 4**: Collaborative budget tracking for group trips.

### Phase 2: Convergent Thinking

**Criteria**:

1. Technical Feasibility
2. User Engagement
3. Development Speed

| Idea | Feasibility | Engagement | Speed | **Total** |
| :--- | :---: | :---: | :---: | :---: |
| AR Graffiti | 1 | 5 | 1 | **7** |
| **Collaborative Budget** | **5** | **4** | **5** | **14** |
| Shared Bucket List | 4 | 3 | 4 | **11** |

### 🏆 Final Selection: Collaborative Budget Tracking

**Rationale**: While AR Graffiti is exciting, it is technically too complex for $V1$. Budget tracking solves a high-priority pain point for travelers and can be implemented quickly using existing database structures.
