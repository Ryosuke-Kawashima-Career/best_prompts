---
description: Update specification documents based on new requirements
---

1. **Analyze Requirements**: Analyze the newly provided requirements or changes from the user. Ask clarifying questions if the scope is ambiguous.
2. **Identify Target Documents**: Review the workspace to identify existing specification documents (requirements, implementation plans, reports, walkthroughs) that are impacted.
3. **Apply Skill**: Use the `spec-update` skill to draft the necessary modifications to these documents. Ensure the standard structure (Overview, Prerequisites, Step-by-Step Guide, Expected Outcomes, Troubleshooting) is maintained where appropriate.
4. **Present Updates**: Provide the updated documents to the user for review and approval.
