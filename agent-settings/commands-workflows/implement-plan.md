# Implement Plan

## Overview

The aim of this process is to write the actual code based on the implementation plan documents, referring to user stories and functional requirements.
The walkthrough document should also be written so that the chain of thought and the reasons of the implementation are comprehensible.

## Format

- The implementation plans are in ./specs/plan.md, which can sometimes be provided by the user instead.
- The walkthrough should be in ./docs/plan.md, which can sometimes be assigned by the user instead.
- The results should be organized for each step of the implementation plan.
- The way to run the code and check the validity of the implementation should be on the walkthrough.
- Show the contents of the test and the outcomes of the test.
